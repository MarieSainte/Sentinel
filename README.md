# Sentinel

Sentinel is an investigation assistant designed to help journalists, researchers, and investigators turn complex information into structured, searchable knowledge.

The project explores a simple idea: people should be able to describe facts, leads, documents, names, events, and relationships in natural language, then let the system help organize that information across the right kinds of databases.

## Why Sentinel Exists

Investigations often involve scattered facts: a name in a document, a payment mentioned in a testimony, a location repeated across reports, or a weak signal that only becomes meaningful later.

Sentinel aims to make those connections easier to capture and revisit. Instead of forcing investigators to immediately choose a rigid form or database model, the interface starts with natural language and helps transform information into usable structures.

## What Sentinel Is Built For

- Capturing new facts, leads, and notes in multiple languages
- Classifying information by reliability and available evidence
- Helping identify people, organizations, places, documents, and events
- Preparing information for structured, graph, and vector databases
- Supporting faster links between facts across an investigation
- Making reports easier to build from verified and traceable information

## Reliability Levels

Sentinel is designed around the idea that not all information has the same weight. Early leads, plausible claims, and evidence-backed facts should not be treated equally.

The current interface uses three reliability levels:

- **Evidence-backed**: information supported by a source, document, testimony, or trace
- **Plausible, needs verification**: coherent information that still requires confirmation
- **Weak signal**: a fragile lead, isolated clue, or low-confidence signal

This helps investigators keep useful leads without confusing them with confirmed facts.

## Current Focus

Sentinel is currently focused on the frontend experience: creating a clear, professional workspace where investigators can enter information, review suggested structure, and prepare data for later use in reports and databases.

The long-term goal is to connect this experience to multilingual natural-language processing, database query generation, entity linking, evidence management, and intelligent report writing.

## Vision

Sentinel is not meant to replace investigative judgment. It is meant to reduce friction around organization, retrieval, and connection-making, so investigators can spend more time asking better questions and less time reshaping information by hand.

The project is in an early stage, but its direction is clear: build a practical tool for turning messy investigative material into structured, traceable, and reusable knowledge.
